Cloud support

FAQ

Q) Why was the CCoE (Cloud Centre of Excellence) created?
A) This platform has been created by the Cloud Vending Team to replace their confluence area. The CCoE will help users find cloud guidance, book cloud training, and use the cloud services.
Q) How do I get extra support working on the cloud?
A) Depending on what support you need you can find information on the CCoE on Cloud Vending Team Support, AWS Support, and Microsoft Azure Support.

Cloud training

Q) How do I know what AWS training to go for?
A) See the Role based training page within the CCoE. It provides detail about the different job families and will help direct you to what is the best training for you.
Q) How do I know what Azure training to go for?

A) The best way to work out what Azure training to take is to visit their training platform Enterprise Skills Initiative (ESI). You can also visit our confluence page where we have interactive PDF's that allow you to explore recommended learning pathways based on your needs.

There is a dedicated page on the CCoE telling you what ESI is, how to sign up, and provides links to videos on how to use the Learner Experience Portal (LxP).

Q) How do I book AWS/Azure training?

A) All bookings are self-service.

See the Cloud Training section on the CCoE for all the information you need to book. The side navigation shows what pages are for AWS (Amazon Web Services) and for Azure.

Q) How do I register for AWS certifications?
A) See the AWS certifications page on the CCoE for all the steps you need to take.
Q) How do I register for Azure certifications?
A) See the Azure certifications page on the CCoE for all the steps you need to take. There are different steps to take depending on if you are going to use your POISE account or a digital device.

Cloud guidance

Q) What are cloud pillars and the well-architected framework?

A) Cloud providers have developed an ideal framework to build cloud applications. The framework is both a set of guidelines and a mechanism for assessing your work against best practices. It has been designed based on aligned and connected pillars. These pillars help cloud architects to create a secure, high-performing, resilient and efficient infrastructure.

The cloud guidance pages on the CCoE provide more information and links.

Q) What are tagging standards and how do I use them?
A) There is a detailed page on the CCoE on what tagging standards are, best practices, current problems, tag keys and terraform examples.
Q) What are access keys?

A) AWS access keys are long-term credentials for IAM (Identity and Access Management) users. Access keys consist of an ID and a secret and provide programmatic access. Programmatic access is for services access, for example the S3 Bucket. Programmatic access is used to automate processes and actions to allow services to interact with Amazon. Access keys can be used to sign programmatic requests to the AWS Command Line Interface (CLI) or AWS Application Programming Interface (API).

There is a dedicated page on access keys within the CCoE.

Q) How do I rotate my access keys?

A) Access keys must be rotated at least every 90 days. The Cloud Vending Team do carry out periodic audits and keys older than 90 days could result in your account being suspended.

Follow the AWS step-by-step guidance for rotating access keys.

The CCoE does provide more information on what access keys are, the policy behind them and how to use access keys in different AWS services.

Q) How do I find out more information on cloud budgets?
A) There is a detailed page on the CCoE on recharging in the Home Office, why we recharge, AWS cost and recharge breakdown, useful links, and links to cloud costs calculators.
Q) How do I change my AWS MFA (Multi Factor Authentication)?

A) You cannot change an existing AWS MFA or remove it yourself.

You first need to put in a request to Cloud Vending for your existing AWS MFA to be removed. You do this by raising a ticket on the IAM Kanban board. There are instructions on how to create an IAM request if required.

Once Cloud Vending have removed your AWS MFA you can set up a new one. AWS have detailed information available on how to enable MFA for devices if required.

Cloud access

Q) How do I raise an IAM ticket?
A) You raise an IAM ticket on the Cloud Vending Kanban board. There are instructions available on the CCoE on how to create an IAM request if required. There are a few steps to take but if you are unsure, reach out to the Cloud Vending Team emailing public.cloud@homeoffice.gov.uk
Q) Who can approve an IAM request?
A) Only individuals on our approvers list can approve an IAM request. Depending on which account you are looking to gain access to, depends on which approver you need to seek comment from. You must tag the approver(s) in your ticket so they can provide comment.
Q) What is a Sandbox environment?

A) A sandbox is a copy of your production environment. The changes you make in the sandbox does not affect your actual production environment. You can use sandboxes to make changes in a test environment and experiment without any consequences.

There is also information on sandbox best practices available within the CCoE. It is important to familiarise yourself with these.

Q) How do I get access to a Sandbox?
A) The CCoE holds the information on how to get access to sandbox accounts, as well as a list of the Home Office sandbox accounts.
Q) What is the public cloud password policy?
A) The Public cloud password policy explains password requirements and how to reset your password.

Cloud support

Q) How do I raise an AWS support case?
A) The AWS support page explains how to raise an AWS support case and information on Enterprise Support.
Q) How do I raise a Microsoft Azure support case?
A) The Microsoft Azure support page explains how to raise an Azure support case and information on Unified Support.
