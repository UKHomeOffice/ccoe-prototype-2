What is Amazon S3

Amazon Simple Storage Service (Amazon S3) is storage for the Internet. It is designed to make web-scale computing easier. Amazon S3 has a simple web services interface that you can use to store and retrieve any amount of data, at any time, from anywhere on the web. It gives any developer access to the same highly scalable, reliable, fast, and inexpensive data storage infrastructure that Amazon uses to run its own global network of web sites. The service aims to maximize benefits of scale and to pass those benefits on to developers.

Configuration of S3 Buckets

Here you will find all information relating to good practice configuration of S3 buckets on AWS public cloud infrastructure. The patterns are recommended to be applied to any S3 instance resident across Home Office platforms.

We recommend talking with your HOCS Assurer and Security Team about the configuration and this page can be used to inform permission to create and configure S3 buckets. This page adapts the AWS best practices and the work HMPO security have done on defining the secure configuration of S3.

Production best practices

Production best practices
Practice Advice Possible variations
Use Amazon S3 block public access Must Exception would be if public access is needed (i.e. for CloudFront) with other security controls in place, e.g. allowlisting, write control etc.
Identify Amazon S3 bucket policies that allow a wildcard identity Must N/A
S3 Bucket Permissions should not be flagged in Trusted Advisor Should Specific exceptions documented here
Use IAM roles for applications and AWS services that require Amazon S3 access Must N/A
Access Keys haven't been created to access the bucket Should Access keys may be used with appropriate authorisation. These must be rotated every 90 days in line with good practice
MFA Delete is enabled on the CloudTrail bucket Must MFA delete is recommended for other buckets too
Bucket is Encrypted at Rest Must All data must be encrypted. Choice of methodology is dependent on data sensitive, but majority of use cases can use KMS sourced AWS-generated keys
aws:SecureTransport condition on Amazon S3 bucket policies Must Only exception if technology is not supported, but steps should be taken to address deficiency
S3 Object Lock enabled Must Data not operating under write-once, read-many (WORM) model, prevents deletion/overwrite
S3 Versioning enabled Must If agreed with risk owner and risk assurer - services may operate a similar function by writing new object to a bucket rather than new version of the object
Amazon S3 cross-region replication enabled Should Often HO data cannot be offshored as back-up to another region, so where this is the case cross-region replication should be disabled. Where data can be offshored (e.g. to Ireland) cross-region replication should be enabled in line with AWS good practice.
Use of VPC endpoints for Amazon S3 access Should Dependent on architecture, so use case may not be scoped within VPC
CloudWatch metrics for Amazon S3 Should If team does not have capability to monitor and this is agreed with risk owner and risk assurer
Enable Amazon S3 server access logging Must If agreed by the risk owner
AWS Config enables you to assess, audit, and evaluate the configurations of your AWS resources Should Where additional cost of Config would be prohibitive for service usage
Metadata attached to objects not sensitive Must N/A
Deploy tags for S3 buckets and objects Should N/A
