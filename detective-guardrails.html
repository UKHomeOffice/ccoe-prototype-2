Detective guardrails

This page is for Amazon Web Services (AWS) Identity and Access Management (IAM) Home Office cloud users who want to understand what alerting we apply through AWS Guardrails in AWS accounts.

This content explains:

  • what detective guardrails are
  • how to implement detective guardrails
  • OU mapping
  • monitoring detective guardrails
  • cost of detective guardrails
  • proposed and future guardrails
Back to side navigation Back to side navigation

Detective guardrails

A detective guardrail detects non-compliant resources within accounts. Alerts are then sent to a dashboard with guardrails either being compliant, non-compliant or not enabled. A detective guardrail does not prevent or remediate non-compliant resources. It will simply flag non-compliance. Detective guardrails are implemented using AWS Config and AWS Lambda.

Back to side navigation

Implement detective guardrails

To implement detective guardrails, you need AWS Config and an active Config recorder in each AWS account. You can then implement the rules in one of two ways.

  • Deploy AWS Config rules to accounts through a service such as AWS CloudFormation aligned to specific OUs.
  • Use AWS Security Hub to act as a central dashboard, with each AWS account added as a child account within Security Hubs.

Security Hub comes with security standards that can be deployed for both the Center for Internet Security (CIS) AWS Foundation Benchmark and AWS Foundational Security Best Practices standard. This automatically deploys the relevant Config rules and other checks that extend beyond those Config can achieve on its own. These are deployed into child accounts and Security Hub provides a central dashboard for visibility. Security Hub also integrates the findings from other products including GuardDuty operating in child accounts.

Back to side navigation

OU mapping

Guardrails can be applied to either individual accounts, or AWS organisations OUs. Applying them to individual OU's simplifies management because they also apply to all child accounts under the OU. It also allows the boundary of guardrail groups to be easily identified.

Mapping guardrails to OUs also allows groups of accounts to have stricter or more relaxed governance applied to them. For example, accounts under a production OU may have stricter compliance requirements than sandbox accounts under a sandbox OU. Eventually each of the guardrails identified will be mapped to individual OUs.

Back to side navigation

Monitoring

Detective guardrails are the same if you implement them using AWS Config or AWS Security Hub. They require a team to review the results and take actions from them. In the future they must implement reactive security controls and automation.

Back to side navigation

Costs

Due to detective guardrails use of AWS Config, they do have costs associated with them. So, there will be additional charges in each account. There are charges for each configuration item recorded per account, per month and the number of config rule evaluations undertaken on resources per month. The total charges per account will depend upon the number of resources and config rule evaluations undertaken.
